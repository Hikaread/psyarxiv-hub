Press Ctrl+C for a graceful shutdown; restart to resume from where you left off.</li>\n<li>📡 <strong>Streaming Mode</strong>: Stream scraped items as they arrive via <code>async for item in spider.stream()</code> with real-time stats - ideal for UI, pipelines, and long-running crawls.</li>\n<li>🛡️ <strong>Blocked Request Detection</strong>: Automatic detection and retry of blocked requests with customizable logic.</li>\n<li>🤖 <strong>Robots.txt Compliance</strong>: Optional <code>robots_txt_obey</code> flag that respects <code>Disallow</code>, <code>Crawl-delay</code>, and <code>Request-rate</code> directives with per-domain caching.</li>\n<li>🧪 <strong>Development Mode</strong>: Cache responses to disk on the first run and replay them on subsequent runs - iterate on your <code>parse()</code> logic without re-hitting the target servers.</li>\n<li>📦 <strong>Built-in Export</strong>: Export results through hooks and your own pipeline or the built-in JSON/JSONL with <code>result.items.to_json()</code> / <code>result.items.to_jsonl()</code> respectively.</li>\n</ul>\n<h3 id=\"advanced-websites-fetching-with-session-support\">Advanced Websites Fetching with Session Support<a class=\"headerlink\" href=\"https://scrapling.readthedocs.io/en/latest/index.html#advanced-websites-fetching-with-session-support\" title=\"Permanent link\">¶</a></h3>\n<ul>\n<li><strong>HTTP Requests</strong>: Fast and stealthy HTTP requests with the <code>Fetcher</code> class. Can impersonate browsers' TLS fingerprint, headers, and use HTTP/3.</li>\n<li><strong>Dynamic Loading</strong>: Fetch dynamic websites with full browser automation through the <code>DynamicFetcher</code> class supporting Playwright's Chromium and Google's Chrome.</li>\n<li><strong>Anti-bot Bypass</strong>: Advanced stealth capabilities with <code>StealthyFetcher</code> and fingerprint spoofing. Can easily bypass all types of Cloudflare's Turnstile/Interstitial with automation.</li>\n<li><strong>Session Management</strong>: Persistent session support with <code>FetcherSession</code>, <code>StealthySession</code>, and <code>DynamicSession</code> classes for cookie and state management across requests.</li>\n<li><strong>Proxy Rotation</strong>: Built-in <code>ProxyRotator</code> with cyclic or custom rotation strategies across all session types, plus per-request proxy overrides.</li>\n<li><strong>Domain &amp; Ad Blocking</strong>: Block requests to specific domains (and their subdomains) or enable built-in ad blocking (~3,500 known ad/tracker domains) in browser-based fetchers.</li>\n<li><strong>DNS Leak Prevention</strong>: Optional DNS-over-HTTPS support to route DNS queries through Cloudflare's DoH, preventing DNS leaks when using proxies.</li>\n<li><strong>Async Support</strong>: Complete async support across all fetchers and dedicated async session classes.</li>\n</ul>\n<h3 id=\"adaptive-scraping-ai-integration\">Adaptive Scraping &amp; AI Integration<a class=\"headerlink\" href=\"https://scrapling.readthedocs.io/en/latest/index.html#adaptive-scraping-ai-integration\" title=\"Permanent link\">¶</a></h3>\n<ul>\n<li>🔄 <strong>Smart Element Tracking</strong>: Relocate elements after website changes using intelligent similarity algorithms.</li>\n<li>🎯 <strong>Smart Flexible Selection</strong>: CSS selectors, XPath selectors, filter-based search, text search, regex search, and more.</li>\n<li>🔍 <strong>Find Similar Elements</strong>: Automatically locate elements similar to found elements.</li>\n<li>🤖 <strong>MCP Server to be used with AI</strong>: Built-in MCP server for AI-assisted Web Scraping and data extraction. The MCP server features powerful, custom capabilities that leverage Scrapling to extract targeted content before passing it to the AI (Claude/Cursor/etc), thereby speeding up operations and reducing costs by minimizing token usage. (<a href=\"https://www.youtube.com/watch?v=qyFk3ZNwOxE\">demo video</a>)</li>\n</ul>\n<h3 id=\"high-performance-battle-tested-architecture\">High-Performance &amp; battle-tested Architecture<a class=\"headerlink\" href=\"https://scrapling.readthedocs.io/en/latest/index.html#high-performance-battle-tested-architecture\" title=\"Permanent link\">¶</a></h3>\n<ul>\n<li>🚀 <strong>Lightning Fast</strong>: Optimized performance outperforming most Python scraping libraries.</li>\n<li>🔋 <strong>Memory Efficient</strong>: Optimized data structures and lazy loading for a minimal memory footprint.</li>\n<li>⚡ <strong>Fast JSON Serialization</strong>: 10x faster than the standard library.</li>\n<li>🏗️ <strong>Battle tested</strong>: Not only does Scrapling have 92% test coverage and full type hints coverage, but it has been used daily by hundreds of Web Scrapers over the past year.</li>\n</ul>\n<h3 id=\"developerweb-scraper-friendly-experience\">Developer/Web Scraper Friendly Experience<a class=\"headerlink\" href=\"https://scrapling.readthedocs.io/en/latest/index.html#developerweb-scraper-friendly-experience\" title=\"Permanent link\">¶</a></h3>\n<ul>\n<li>🎯 <strong>Interactive Web Scraping Shell</strong>: Optional built-in IPython shell with Scrapling integration, shortcuts, and new tools to speed up Web Scraping scripts development, like converting curl requests to Scrapling requests and viewing requests results in your browser.</li>\n<li>🚀 <strong>Use it directly from the Terminal</strong>: Optionally, you can use Scrapling to scrape a URL without writing a single line of code!</li>\n<li>🛠️ <strong>Rich Navigation API</strong>: Advanced DOM traversal with parent, sibling, and child navigation methods.</li>\n<li>🧬 <strong>Enhanced Text Processing</strong>: Built-in regex, cleaning methods, and optimized string operations.</li>\n<li>📝 <strong>Auto Selector Generation</strong>: Generate robust CSS/XPath selectors for any element.</li>\n<li>🔌 <strong>Familiar API</strong>: Similar to Scrapy/BeautifulSoup with the same pseudo-elements used in Scrapy/Parsel.</li>\n<li>📘 <strong>Complete Type Coverage</strong>: Full type hints for excellent IDE support and code completion. The entire codebase is automatically scanned with <strong>PyRight</strong> and <strong>MyPy</strong> with each change.</li>\n<li>🔋 <strong>Ready Docker image</strong>: With each release, a Docker image containing all browsers is automatically built and pushed.</li>\n</ul>\n<h2 id=\"star-history\">Star History<a class=\"headerlink\" href=\"https://scrapling.readthedocs.io/en/latest/index.html#star-history\" title=\"Permanent link\">¶</a></h2>\n<p>Scrapling’s GitHub stars have grown steadily since its release (see chart below).</p>\n<div id=\"chartContainer\">\n  <a href=\"https://github.com/D4Vinci/Scrapling\">\n    <img id=\"chartImage\" alt=\"Star History Chart\" loading=\"lazy\" src=\"https://api.star-history.com/svg?repos=D4Vinci/Scrapling&amp;type=Date\" height=\"400\">\n  </a>\n</div>\n\n<script>\nconst observer = new MutationObserver((mutations) => {\n  mutations.forEach((mutation) => {\n    if (mutation.attributeName === 'data-md-color-media') {\n      const colorMedia = document.body.getAttribute('data-md-color-media');\n      const isDarkScheme = document.body.getAttribute('data-md-color-scheme') === 'slate';\n      const chartImg = document.querySelector('#chartImage');\n      const baseUrl = 'https://api.star-history.com/svg?repos=D4Vinci/Scrapling&type=Date';\n\n      if (colorMedia === '(prefers-color-scheme)' ?
